A factor of 1e9 to convert meter into nanometer is embedded internally in the program SAC (101.6a; 102.0), not in the response files uproaded. Try to find 'unitScaleFact' in the source code of SAC (e.g., grep -ri 'unitScaleFact' ./*). This is, I think, confusing when you use RESP or SACPZ, formats of the response files, interchangeably. Using the SACPZ format does not behave like that. See also SAC manual for more information.
